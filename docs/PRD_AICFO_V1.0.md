## 一、产品概述

### 1.1 产品定位

**一句话定位**：每个小老板都有一个深度AI CFO，"会说话就能管财税"。

AICFO 是一款面向 1-20人小微企业的微信小程序，以 AI 对话为核心交互方式，帮助老板零门槛完成记账、报销、报税、合规等财税管理全流程。

### 1.2 目标用户画像

| 维度 | 描述 |
|------|------|
| 规模 | 1-20人小微企业 |
| 老板特征 | 手机重度依赖，多平台混合经营，不懂财税 |
| 痛点 | 不会做账、不会报税、票据丢失、合规风险 |
| 核心诉求 | 简单、实时、不踩坑 |

### 1.3 产品价值主张

| 价值点 | 说明 |
|--------|------|
| **零门槛** | 会说话就能管财税，无需财税知识 |
| **实时** | 凭证入账即触发报表更新，毫秒级 |
| **合规** | 税务日历预警，发票自动校验，档案合规管理 |
| **智能** | AI意图识别，自动生成凭证、算税草稿 |

## 二、冷启动流程

### 2.1 首次使用路径（4步）

```
Step 1：输入统一社会信用代码
    ↓
Step 2：企业信息自动填充（企查查/天眼查API）
    ↓
Step 3：选择初始化方式（空账 / 旧系统导入 / 银行流水导入）
    ↓
Step 4：激活展示（凭证草稿 + AI使用示例）
```

### 2.2 Step 1 — 信用代码输入

**必填项**：
- [ ] 统一社会信用代码（18位，支持扫码输入）

**企业类型**（必填，影响后续计算逻辑）：
- [ ] 小规模纳税人
- [ ] 一般纳税人

**自动填充项**（输入代码后自动填充，可修改）：
- 企业名称、法定代表人、注册资本及币种、成立日期
- 注册地址（精确到区）、经营范围、纳税人类别预判
- 银行开户行信息（如API支持）

### 2.3 Step 3 — 初始化方式

| 方式 | 说明 |
|------|------|
| **A. 从零开始** | 直接进入日常记账模式，AI引导逐月录入 |
| **B. 旧系统导入** | 下载Excel模板 → 从用友/金蝶导出 → 上传 → AI识别字段映射 → 预览 → 确认导入 |
| **C. 银行流水导入** | 授权微信支付/支付宝商家API → 自动拉取近12个月流水 → AI生成凭证草稿 → 老板逐月确认 |

## 三、功能需求

### 3.1 P0 功能（首个版本必须实现）

#### P0-1：AI对话记账

**触发方式**：用户输入文字 / 语音 / 拍照发票

**对话→凭证矩阵**：

| 用户输入示例 | AI动作 | 输出 |
|-------------|--------|------|
| "收了XX公司3万设计费" | 收入记账 | 生成凭证 |
| "付了京东货款2680元" | 付款记账 | 生成凭证 |
| "帮员工发了工资每人1万" | 调起算薪流程 | 工资单预览 |
| "我要报Q1所得税" | 生成报税草稿 | PDF导出 |
| "看看这个月赚了多少" | 生成月度利润表 | 报表展示 |
| "供应商李总催款，付了他5000" | 付款记账+合同联动 | 更新合同付款记录 |
| "公户转了我私人卡2万" | 弹出三选一处理 | 引导选择 |

**AI对话上下文记忆**：
- 支持"那笔"等代词指代消解（查询最近5笔交易）
- 多轮对话状态保持：sessionId / enterpriseId / currentTaxPeriod / currentFocus

#### P0-2：智能报税合规

| 功能 | 说明 |
|------|------|
| **税务日历** | 自动同步注册地申报日历，提前7/3/1天三级提醒 |
| **一键算税** | 自动区分小规模/一般纳税人，应用对应税率和免税额度 |
| **合规预警** | 🚨紧急（申报截止≤3天）/ 🔴警告（支出无票）/ ⚠️提示（税负率）/ 💡洞察（进项偏低） |
| **申报草稿** | AI生成 → 老板确认 → 导出PDF → 引导至电子税务局 |

#### P0-3：人工专家兜底

**触发条件**（满足任一）：
- AI置信度 < 0.5
- 单笔金额 > 50万
- 连续2次无法理解用户意图
- 用户主动说"转人工"

### 3.2 P1 功能（V1.x迭代实现）

#### P1-1：工资智能算薪

**支持工资类型**：
基本工资 / 绩效工资 / 加班费 / 餐补交通补（免税额度内）/ 全勤奖 / 生育津贴 / 工伤津贴 / 病假工资 / 年终奖 / 离职补偿金 / 防暑降温费 / 高温津贴 / 通讯补贴

**生育津贴专项**：
- 情形A（津贴 > 产假工资）：公司转付全部津贴，员工多得部分按偶然所得，自动计算差额个税
- 情形B（津贴 < 产假工资）：公司补差，补差部分合并计税

**年终奖两种方案自动对比**：
- 方案A：年终奖单独计税（÷12查月度税率表）
- 方案B：合并入当月工资计税
- AI推荐更优方案，显示节税金额

#### P1-2：合同支付管理

| 类型 | 内容 |
|------|------|
| 采购合同 | 供应商 / 金额 / 付款节点 / 开票要求 → 到期提醒 → 付款凭证 |
| 销售合同 | 客户 / 金额 / 收款节点 / 开票税率 → 履约跟踪 → 收款确认 |

**到期提醒**：7天 / 3天 / 当天 / 逾期，四级提醒

**合同税率联动**：

| 场景 | 税率 | 印花税 | 开票时点 |
|------|------|--------|---------|
| 销售商品 | 13%（一般）/ 1%（小规模） | 万分之三 | 收款后 |
| 设计咨询 | 6% | 千分之一 | 履约后 |
| 建筑安装 | 9% | 万分之三 | 按进度 |
| 技术转让 | 6%（可申请免税） | 千分之三 | 完成后 |

#### P1-3：发票OCR采集（V1.3重点）

**支持的发票类型**（共12种）：

| 类型 | 说明 |
|------|------|
| 增值税专用发票 | 可抵扣进项税 |
| 增值税普通发票 | 不可抵扣 |
| 增值税电子专票/普票 | 数电票（OFD格式） |
| 通用机打发票 | — |
| 定额发票 | 停车票/定额餐饮票 |
| 出租车票 | — |
| 火车票 | — |
| 航空行程单 | — |
| 收据 | 仅内部参考 |

**OCR识别5步流程**：

```
Step 1：拍照/上传
    ↓
Step 2：调用OCR（阿里云发票专用API，<2秒响应）
    ↓
Step 3：结构化数据提取
    发票类型/号码/日期/购买方/销售方/金额/税率/税额/商品明细/校验码
    ↓
Step 4：AI智能处理
    判断类型→推荐科目→判断抵扣资格→填入凭证字段→关联对话
    ↓
Step 5：用户确认 → 生成凭证
```

**OCR识别后7项自动关联**：

| # | 关联动作 |
|---|---------|
| ① | 自动与对话关联（填入摘要字段） |
| ② | 自动查重（与历史发票号码比对） |
| ③ | 自动匹配合同（识别销售方→搜索合同台账→询问是否关联） |
| ④ | 自动判断税务抵扣资格（专票+一般纳税人→显示可抵扣金额） |
| ⑤ | 自动生成凭证草稿 |
| ⑥ | 进项税额归集（一般纳税人，自动归入可抵扣进项税台账） |
| ⑦ | 缺失信息预警（购买方税号不符 / 金额>1万无合同） |

**OCR准确率保障**：
- 专票约98% / 普票约97% / 行程票约95% / 手写票约85%
- 低于90%置信度字段标红提示核对
- 关键字段（金额/税号）双重校验（OCR+规则校验）
- 识别失败不阻断，手动输入兜底

## 四、凭证合规体系

### 4.1 凭证9要素（依据《会计基础工作规范》第55条）

| # | 要素 | 规则 |
|---|------|------|
| 1 | 凭证日期 | 必填，自动 |
| 2 | 凭证编号 | 自动格式：YYYYMM-类型-流水号 |
| 3 | 摘要 | AI自动生成，可修改 |
| 4 | 借贷方向 | AI自动判断 |
| 5 | 会计科目 | AI推荐，白名单校验 |
| 6 | 金额 | 精确到分 |
| 7 | 附件数量 | 必填 |
| 8 | 制单人签章 | 系统自动 |
| 9 | 审核人签章 | 用户确认后 |

### 4.2 凭证状态机

```
草稿 → 待确认 → 已入账 → 已申报 → 已归档
```

- 已入账凭证可作废（冲销）或待补充（缺信息）
- 已申报凭证仅人工专家可修改

### 4.3 凭证修改机制

| 修改类型 | 场景 | 处理方式 |
|---------|------|---------|
| **A类：信息勘误** | 已入账，发现填写错误 | 红字凭证冲销 + 蓝字更正凭证，同时展示，老板确认后入账 |
| **B类：跨期调整** | 上期发现本期错误 | 判断影响范围→生成调整凭证，影响以前年度生成以前年度损益调整，同步提示是否需要更正申报，触发人工专家 |
| **C类：发票作废红冲** | 同一月份直接作废重开；跨月已认证申请红字发票→税局审核→重开 | 自动生成红字凭证 |

### 5.1 小规模 vs 一般纳税人差异

| 维度 | 小规模纳税人 | 一般纳税人 |

| 季度免税额度 | 季度30万内普票免征 | 无免征 |
| 申报频率 | 季度申报增值税 | 月度申报增值税 |
| 附加税基数 | 减免后余额×12% | 实际增值税×12% |

| 六税两费减半 | 小微企业 |

## 七、财务报表

### 7.1 实时生成逻辑

**触发时机**：任意凭证确认入账 → 立即触发三大报表重算（毫秒级）

| 报表 | 计算逻辑 |
|------|---------|
| **资产负债表** | 资产总计=Σ资产类借方余额；负债总计=Σ负债类贷方余额；所有者权益=Σ权益类贷方余额；恒等式自动校验 |
| **利润表** | 本月数据=Σ损益类本月发生额；净利润=收入-成本-费用+营业外收支 |
| **现金流量表** | 经营现金流=银行流水（主要来源）+AI根据凭证类型自动归类，生成草稿，老板核对 |
