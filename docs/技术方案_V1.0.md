## 一、技术选型

### 1.1 小程序框架

| 选项 | 推荐指数 | 说明 |
|------|---------|------|
| **原生微信小程序** | ⭐⭐⭐⭐⭐ | 性能最优，包体积最小，微信生态深度集成，发布流程成熟 |
| Taro 3.x/4.x（React） | ⭐⭐⭐ | 跨端支持好，但包体积增加约30%，首屏略慢 |
| uni-app（Vue） | ⭐⭐⭐ | 跨端强，微信小程序端不如原生稳定 |

**推荐方案：原生微信小程序 + TypeScript + Vite 构建**

- 开发语言：TypeScript（类型安全，利于多人协作）
- UI 组件库：Vant Weapp（有赞，组件丰富，体积可控）
- 状态管理：Pinia（小程序适配版）或 MobX-mini
- 构建优化：Vite + miniprogram-compiler（提升构建速度）
- 语音输入：微信同声传译插件（实时语音转文字）
- 本地存储：wx.openDatabase（SQLite，离线草稿缓存）

### 1.2 后端技术栈

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| **Web 框架** | FastAPI（Python 3.11+） | 高性能 ASGI，支持异步，自动 OpenAPI 文档，类型提示完善 |
| **ORM** | SQLAlchemy 2.0 + Alembic | 支持异步，迁移管理成熟 |
| **异步任务** | Celery + Redis | 报表生成、批量导入、批量推送等 |
| **AI 服务层** | LangChain + LangServe | AI 编排框架，支持多模型切换和降级 |
| **Web 服务** | Uvicorn（ASGI） | 高性能 Python ASGI 服务器 |
| **API 网关** | Nginx | HTTPS 终结，静态资源，限流 |
| **容器化** | Docker + Docker Compose | 开发/生产一致性部署 |

### 1.3 数据库

| 数据库 | 选型 | 说明 |
|--------|------|------|
| **主数据库** | PostgreSQL 15+ | 强事务 ACID，JSONB 支持半结构化，GIS 扩展 |
| **缓存层** | Redis 7 | Session、Token、限流计数、AI 对话上下文缓存 |
| **全文检索** | Meilisearch（可选） | 发票台账、合同文本模糊检索 |
| **时序/分析** | PostgreSQL + TimescaleDB（可选） | 财务报表时序聚合查询 |

### 1.4 部署方案

| 环境 | 方案 | 说明 |
|------|------|------|
| **开发环境** | Docker Compose 本地 | PostgreSQL / Redis / FastAPI 一键启动 |
| **测试环境** | 阿里云 ECS（2核4G × 2台） | 一台 Web，一台 DB，SLB 负载 |
| **生产环境** | 阿里云容器服务 ACK（ASK） | 自动弹性扩缩容 |
| **CI/CD** | GitHub Actions + 阿里云 ACR | 代码推送自动构建 Docker 镜像并部署 |
| **域名/证书** | 阿里云 DNS + SSL 免费证书 | HTTPS 全站加密 |

**MVP 阶段成本预估（月）**：
- 阿里云 ECS 2台：约 ¥400
- OSS 存储（发票/凭证附件）：约 ¥20
- Redis 云服务：约 ¥30
- 短信/推送：约 ¥50
- **合计：约 ¥500/月**

### 2.2 核心请求链路：AI对话记账

```
① 用户输入（文字/语音/拍照发票）
      ↓
② 微信小程序 → POST /api/v1/chat
      ↓
③ Nginx（IP限流 → JWT校验 → 请求路由）
      ↓
④ FastAPI Chat API（挂起等待AI响应）
      ↓
⑤ Redis（读取 sessionId 对应上下文缓存）
      ↓
⑥ DeepSeek V3.1 API
    Intent识别（income_record / expense / tax_query...）
    凭证生成（根据意图构建分录）
      ↓ 限流429
⑦ 通义千问2.5 API（降级，回复标注"当前服务繁忙"）
      ↓ 全部失败
⑧ 返回友好错误 + 保存凭证草稿（PostgreSQL）+ 转人工入口前置
      ↓
⑨ AI置信度 < 0.5 → 自动创建专家工单
      ↓
⑩ WebSocket 推送小程序（凭证草稿流式展示）
      ↓
⑪ 小程序展示凭证预览 → 用户确认/修改
      ↓
⑫ POST /api/v1/vouchers/{id}/confirm → 凭证正式入账
      ↓
⑬ Celery 异步任务 → 触发三大报表重算（毫秒级）
```

### 3.3 AI对话模块设计

**模块文件结构**：
```
modules/ai-chat/
├── chatStore.js               # MobX/Pinia 状态管理（会话列表、当前消息）
├── chatService.js             # API调用封装（HTTP轮询 + WebSocket流式）
├── intentEngine.js            # 前端意图解析（降级兜底UI展示）
├── voucherRenderer.js         # 凭证草稿卡片渲染逻辑
├── speechRecognizer.js        # 语音识别封装（微信同声传译）
└── offlineCache.js            # SQLite离线缓存管理
```

**核心对话流程**：
```
1. 用户输入（文字/语音/图片）
   ↓
2. chatStore.addUserMessage() 添加用户消息
   ↓
3. chatService.sendMessage() → POST /api/v1/chat
   同时建立 WSS 连接 /ws/chat/{sessionId}
   ↓
4. WebSocket 流式接收 AI 响应（打字机效果，逐字展示）
   ↓
5. intentEngine.parse() 解析消息类型：
   - type: "voucher_draft"   → voucherRenderer.render() 凭证卡片
   - type: "tax_draft"       → taxPreview.render() 税务草稿卡片
   - type: "clarification"   → 显示AI追问选项按钮
   - type: "text"            → 直接文字展示
   - type: "chart"           → 图表渲染组件
   ↓
6. 用户确认凭证 → POST /api/v1/vouchers/{id}/confirm
   ↓
7. 凭证正式入账 → 本地 SQLite 异步备份（防丢）
```

**代词指代消解策略（前端配合）**：
- 监听"那笔""刚才""上次的"等代词
- 调用 GET /api/v1/vouchers/recent?limit=5 拉取最近5笔
- 将代词替换为具体凭证ID后发送

**WebSocket 连接管理**：
```javascript
// 连接建立
wx.connectSocket({ url: 'wss://api.aicfo.com/ws/chat/' + sessionId })

// 断线自动重连（指数退避：1s → 2s → 4s → 8s → 16s，最多5次）
// 心跳：每30秒发送 ping，60秒无响应断开重连
// 消息格式：{type: "message"|"voucher"|"tax"|"done"|"error", data: {...}}
```

**离线缓存策略**：
```javascript
// 凭证草稿写入本地 SQLite
const db = wx.openDatabase({ name: 'aicfo_offline.db' })
db.execute(`INSERT INTO voucher_drafts (...) VALUES (...)`)

// 网络恢复检测
wx.onNetworkStatusChange((res) => {
  if (res.isConnected) syncDraftsToServer()
})

// 冲突策略：服务器优先，展示冲突提示让用户确认
```

### 7.2 凭证生成逻辑

**Step 1：解析口语输入**
```
"收了XX公司3万设计费"
  ↓
{
  counterparty: "XX公司",
  amount: 30000.00,
  nature: "设计费",      // AI判断
  direction: "income",   // 收款
  has_tax: true          // 一般默认含税
}
```

**Step 2：选择会计科目（白名单内）**
```
收入性质=设计费 → 主营业务收入（6001）
是否含税=是 + 纳税人类型=一般纳税人 → 销项税额（2221）
款项来源=XX公司 → 应收账款（1122）或银行存款（1002）
```

**Step 3：构建分录**
```
借：银行存款          30,000.00
    贷：主营业务收入     28,301.89
        应交税费-销项税额  1,698.11
（税率6%，小规模1%，根据企业类型自动适配）
```

**Step 4：生成摘要**
```
"收到XX公司设计费（含税）"
```

**Step 5：自动关联（若可匹配）**
- 扫描合同台账：若XX公司有未结销售合同 → 询问是否关联合同付款节点
- 发票关联：若用户拍照了发票 → 发票ID写入凭证关联字段

### 7.3 上下文记忆设计

**Redis缓存结构**：
```json
{
  "session:{sessionId}": {
    "enterpriseId": "ent_001",
    "currentTaxPeriod": "2026-03",
    "currentFocus": "income_record",
    "recentVouchers": [
```
